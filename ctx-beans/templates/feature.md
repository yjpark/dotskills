# Feature Template

A feature describes a user-facing capability or enhancement.

## Sections

### Problem / Motivation
What need or pain point does this address? Why does this matter to users?

### Proposed Solution
High-level description of the feature. What is being built?

### User Experience
How do users interact with this? What changes for them day-to-day?

### Acceptance Criteria
Concrete, verifiable conditions for "done". Each criterion should be independently testable.

### Scope & Boundaries
What is explicitly included. What is explicitly out of scope for this feature.

### Dependencies & Constraints
Blockers, technical limitations, related work that must happen first or alongside.
